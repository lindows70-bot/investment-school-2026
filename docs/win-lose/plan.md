# ⚔️ 승패 해부실 (Winners vs Losers) — 설계

## 목적
"지금 이 장에서 오르는 종목과 떨어지는 종목은 뭐가 다른가"를 **매일 자동 갱신**해 보여주는 상설 교육 화면.
개별 종목 판정(WHAT/WHEN)이 아니라 **시장의 채점 기준(요인 로테이션)**을 역으로 관측 — 장이 바뀌면 승부 요인이 바뀌는 걸 학생이 매일 목격하는 게 목적.

## 사용자 확정 사항 (2026-07-14)
- 유니버스 = 학교 보유 합집합 + 스크리너 514종 (`macro-screened-universe:v7` 재사용)
- 배치 = 사이드바 📌매일 그룹(브리핑 바로 아래) 독립 페이지 `/win-lose` + 브리핑에 ⚔️ 요약 섹션

## 설계 원칙 (앱 철학)
- **신규 판정기 0** — 펀더·추세·EPS방향은 ScreenedStock(스크리너 캐시) 재사용, 섹터 국면은 sector-rotation 캐시 읽기만(콜드면 null graceful), 수익률·52주 위치만 Yahoo 차트로 신규 계산(알파헌터 선례: KR도 Yahoo .KS→.KQ)
- **관측이지 추천 아님** — 점수·추천 절대 미반영. "과거 N주의 채점 기준이지 미래 보장 아님" 캐비엇 명시
- **결정론 교훈 생성** — AI 미사용. 요인별 승패 격차(gap 0~100)를 계산해 Top3 + '못 가른 요인'을 문장 자동 조립
- **프라이버시** — 학교 보유 종목은 유니버스에 섞되 '학교 보유' 배지 없음(단독보유 노출 차단). '내 보유' 하이라이트만 클라이언트에서 본인 investments(RLS)로

## 구조
- `src/lib/winLose.ts` (SSOT·순수함수): WLRow 타입 + `splitGroups`(기간별 임계 1w±1.5/1m±3/3m±5) + `factorStats`(7요인 격차) + `buildLesson`(오늘의 교훈 문장)
  - 7요인: ①추세 정배열비율(priceTrend SSOT) ②52주 위치 ③시장(US 비중) ④수익성(영업이익률) ⑤EPS 상향비율(fwdEpsDir) ⑥주도섹터 비율(로테이션 quad) ⑦밸류 PEG
- `/api/win-lose` (공개·`win-lose-v1:KST날짜` 12h·maxDuration 300): 유니버스 조립 → 종목별 Yahoo 차트(동시성 8·~600콜 ≈ 80s) → ret1w/1m/3m·pos52 → 로테이션 quad 조인(SECTOR_TO_ROT — unified-reco와 동일 맵). 성공 60% 미만 캐시 박제 금지
- 크론: 매일 23:50 UTC(08:50 KST — 타점워처 뒤) `/api/win-lose` 워밍
- `/win-lose` 페이지: ①스코어보드+기간토글+교훈배너 ②⚔️대전표(나비 바·못가른 요인 회색) ③🧭섹터 전장 지도(로테이션 국면순) ④🎯산점도(X=52주 위치·Y=수익률·색=추세·내보유 흰테두리) ⑤🏆/💀 Top8 ⑥🎓캐비엇
- 브리핑: ④판읽기 다음 ⚔️ 요약(교훈 1줄+미니 대전표 3행+딥링크)
