# 💸 연말 절세 도우미 (양도세 손익통산) — 설계

> 7대 킬러 기능 로드맵 #1 (2026-07-18). "더 벌게"가 아니라 "확정적으로 잃지 않게" — 12월 절세 매도는 수익률에 확정 +가 되는 몇 안 되는 행동.

## 목적
학생의 올해 **해외주식 확정 양도차익**을 자동 집계하고, 250만원 기본공제 대비 초과분·예상 세금을 보여준 뒤,
- 세금이 나올 상황이면 → **손실 보유 종목 매도(loss harvesting)로 얼마를 절약**할 수 있는지
- 공제 여유가 남으면 → **이익 종목 일부 익절+재매수(gain harvesting)로 취득가를 올려** 미래 세금을 줄일 수 있는지
를 Zero-Input 결정론으로 처방한다. ⛔ 자동매매 없음·세무 상담 아님(교육·참고).

## 과세 규칙 (2026년 기준 — 코드 상수·문구 근거)
| 자산 | 세제 | 이 도우미 |
|---|---|---|
| 해외주식·해외상장 ETF (market=US) | 양도세: 연 **250만원 기본공제**, 초과분 **22%**(지방세 포함). **같은 해 손익통산 가능** | ✅ 핵심 대상 |
| 국내 상장주식 (소액주주 장내) | **비과세** (금투세 폐지 확정 이후 현행 유지) | 참고 표시만 (통산 대상 아님) |
| 국내 상장 해외/파생형 ETF | 매매차익 **배당소득세 15.4%** (보유기간과세·손익통산 불가) | 교육 안내만 |
| 암호화폐 | 과세 **2027년으로 유예** | 안내만 |
- 한국은 **워시세일 규정 없음** → 매도 직후 재매수 가능. 단 재매수 시 취득가 리셋(미래 양도차익 증가) 정직 안내.
- 과세연도 귀속 = **결제일 기준**(미국주식 T+1) → 12월 마지막 거래일 매도는 이듬해 귀속 위험 — 컷오프 날짜 하드코딩 대신 "연말 며칠 여유" 캐비엇.

## 데이터 (전부 기존 재사용 — 외부 신규 수집 0)
- **확정 손익**: `transactions` — type='sell', 올해(transaction_date), `realized_pnl`(⚠️ **종목 통화 기준** — USD는 환산 필요), `currency`·`market`.
- **보유 평가손익**: `investments`(purchase_price·quantity) + `/api/stock-price` POST 배치(현재가) — US만(KR 비과세·CRYPTO 유예라 harvest 무의미).
- **환율**: `/api/exchange-rate` (폴백 1380).

## 계산 (결정론)
```
realizedUsKrw = Σ sell(US).realized_pnl × usdKrw          # 올해 확정(해외)
taxable   = max(0, realizedUsKrw − 2,500,000)
estTax    = taxable × 0.22
room      = max(0, 2,500,000 − realizedUsKrw)             # 남은 비과세 이익 실현 여유
# loss harvesting (taxable>0일 때): 손실 보유 US 종목 —
#   전량 매도 시 실현손실 L → 절약 = min(L, taxable) × 0.22 (절약액 순 정렬)
#   필요 손실 실현액 = taxable (이만큼 실현하면 세금 0)
# gain harvesting (room>0일 때): 이익 보유 US 종목 —
#   여유 내 매도 가능 주수 = floor(room / 주당이익KRW) → 익절+재매수 = 취득가 상향(미래 절세 최대 room×22%)
```

## 정직 캐비엇 (전부 UI 명시)
1. **환율 근사**: 세법상 매수/매도 각각의 **결제일 환율**로 원화 환산 — 우리는 현재 환율 일괄 적용(추정치). 실제 신고는 증권사 '해외주식 양도소득 내역' 기준.
2. **이 앱에 기록된 거래 기준** — 앱 밖 계좌·타 증권사 실현손익은 미반영(공제는 인별 합산).
3. 결제일 컷오프(연말 매도는 여유 있게) / 재매수 시 취득가 리셋 / 세무 상담 아님.
