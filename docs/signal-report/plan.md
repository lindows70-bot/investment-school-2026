# 📋 앱 신호 성적표 (Signal Report Card) — 설계

> 7대 킬러 로드맵 #2 (2026-07-18). "이 앱의 신호를 얼마나 믿어야 하나"를 앱 스스로 데이터로 채점 — 정직 원칙(가짜 승률 금지)의 마지막 퍼즐. 신호 이력이 익어야 하는 기능이라 적립 장치를 일찍 심는 게 핵심.

## 채점 대상 (v1 — 2개 소스)
| 소스 | 이력 | 이벤트 정의 |
|---|---|---|
| 🤖 **Jarvis 모닝 처방전** SELL/BUY | `user_daily_briefings` 테이블에 **2026-06부터 이미 적립** → 즉시 30~45일 채점 가능 | 매일 반복 판정이므로 **연속 런의 첫날 = 1이벤트**(같은 신호 연속은 자기상관 — 신호가 끊겼다(>7일) 재등장하면 새 이벤트) |
| 🚦 **타점 전환 워처** buy/sell | `timing-watch-changes-v2`는 당일분 덮어쓰기 → **`signal-history-v1`에 매일 append**(타이밍 크론 안에서 인라인 적립 — 별도 크론 없이 원자적) | 전환 자체가 이벤트(diff 기반이라 중복 없음) |
| 통합추천 Top12 | v2 보류 — auth·보유 제외가 얽힌 라우트라 엔진 lib 분리 필요(정직 기록) | — |

## 채점 방법 (결정론·AI 미사용)
- **가격 SSOT**: `techChartData.getTechCandles(D)` — 타이밍 크론이 매일 전 보유 종목을 캐시(tech-chart-v1)하므로 추가 부하 ~0. Jarvis 대상 = 보유 종목 부분집합이라 전부 캐시 히트.
- 진입가 = 이벤트 날짜 이하 가장 가까운 종가(신호는 개장 전 발생 — ±1일 오차 캐비엇).
- **+30일 수익률**(이벤트가 30일 이상 익었을 때만) + **현재까지 수익률** 둘 다.
- 승 판정: BUY = 수익률 > 0 / **SELL = 수익률 < 0**("하락 적중" — SELL은 '매도검토' 신호지 공매도가 아님을 UI 명시).
- 그룹: 소스 × 방향 4개 — n·승률·평균 수익률(30일/현재)·베스트/워스트 사례.

## 정직 캐비엇 (UI 명시)
1. 표본이 적으면(특히 <10건) 통계가 아니라 일화 — 표본수 상시 병기.
2. 대상 = 학생 보유 종목뿐(유니버스 아님) → 생존·선택 편향 존재.
3. SELL 채점은 '신호 후 하락 여부'이지 공매도 수익 아님 / 과거 성과 ≠ 미래.
4. 진입가 ±1일 오차(신호 발생 시각 vs 종가) / 배당 미반영.

## 구현
- **크론 수정**: `cron/timing-watch`가 changes 기록 시 `signal-history-v1`(배열·상한 2,000건)에 append — `{date,src:'timing',ticker,market,kind,label}`.
- **`/api/signal-report`**(공개·`signal-report-v1:{KST일자}` 12h·maxDuration 120): Jarvis 런 압축(서비스롤) + 타이밍 이력 → 캔들 채점 → 그룹 통계.
  - 개인정보: 종목 신호만 노출(보유자·인원 무표기) — 승패 해부실 '우리 포트 보드'와 동일 익명 관례.
- **`/signal-report` 페이지**: 사이드바 📌매일 그룹(승패 해부실 아래). 소스별 성적 카드 + 사례 칩 + 적립 현황(타이밍은 "적립 중 D+N") + 캐비엇.
